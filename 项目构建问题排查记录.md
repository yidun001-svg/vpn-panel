# 项目构建问题排查记录

## 1. 对话概述

本次会话专注于解决 **Expo SDK 54 React Native 应用** 在 Android Gradle 编译过程中因缺少 `AnyTypeCache` 类而导致的构建失败问题。该问题表现为运行时 `java.lang.NoClassDefFoundError: expo.modules.kotlin.types.AnyTypeCache`，根本原因是不兼容的 `expo-asset` 版本（57.0.3）被 `expo-audio@56.0.12` 引入。解决方案是通过显式添加 `expo-asset@~12.0.13` 依赖来对齐 SDK 54 的捆绑版本。依赖解析修复后构建成功。

## 5. 解决方案与故障排查

### 问题：运行时找不到 `AnyTypeCache`

- **症状**：当 `AssetModule.definition()` 运行时出现 `java.lang.NoClassDefFoundError: expo.modules.kotlin.types.AnyTypeCache`
- **根本原因分析**：
  1. `expo-asset@57.0.3` 的 AAR 是针对包含 `AnyTypeCache` 的更新版 `expo-modules-core` 编译的
  2. AAR 中 `AssetModule.class` 的 `definition()` 方法使用 `AnyTypeCache` 进行类型缓存：字节码显示 `getstatic #369 // Field expo/modules/kotlin/types/AnyTypeCache.INSTANCE` 和 `invokevirtual #390 // Method expo/modules/kotlin/types/AnyTypeCache.getTypesMap`
  3. 项目使用的 `expo-modules-core@3.0.29`（SDK 54）缺少此类
  4. 这是一个**预编译 AAR 不兼容性问题** — Kotlin 源代码可能不直接引用 `AnyTypeCache`，但字节码中存在（很可能是 Kotlin 编译器对 `Coroutine` 内联闭包模式的优化所致）
- **影响**：Android 构建成功（`BUILD SUCCESSFUL`），但运行时模块加载时崩溃

### 解决方案：版本锁定

1. 确认 `expo-asset@12.0.13`（与 `expo@54.0.33` 捆绑）**不引用** `AnyTypeCache`
2. 在 `package.json` 的 `dependencies` 中添加 `"expo-asset": "~12.0.13"`
3. 运行 `npm install` — 移除 7 个包，更改 3 个包，正确解析
4. 重新构建成功

### 调试方法论

- 使用 `npm pack` 下载并检查多个版本的 AAR 内容
- 使用 `javap` 反编译字节码以确认 `AnyTypeCache` 引用
- 跨版本比较 AAR 内容：12.0.13（兼容）vs 56.0.0 / 57.0.3（不兼容）
- 验证 `expo-modules-core` 3.0.29 和 3.0.30 都缺少 `AnyTypeCache`
